## üì± Google Play Store Screenshot Requirements

### General Requirements

**Phone screenshots:**
- **Minimum:** 2 screenshots required
- **Maximum:** 8 screenshots allowed
- **Formats:** JPEG or 24-bit PNG (no alpha channel)
- **Minimum dimension:** 320 px
- **Maximum dimension:** 3840 px
- **Aspect ratio:** 16:9 to 9:16 (portrait or landscape)

**Tablet screenshots (7-inch):**
- Optional but recommended if app supports tablets
- Same format requirements as phone
- Minimum 7-inch tablet resolution

- [ ] All screenshots same size/aspect ratio
- [ ] Screenshots are in correct order (tell a story)
- [ ] High resolution, no pixelation
- [ ] Realistic but clean sample data
- [ ] No personal/sensitive information visible

- [ ] Consistent app version across all screenshots

## üéØ Best Practices

### DO:
- Show actual app interface
- Use high-quality, crisp images
- Highlight key features first
- Keep text minimal if using overlays
- Use consistent screenshot style
- Show app in action (not just static screens)
- Include compelling visuals

### DON'T:
- Use outdated UI
- Show error states or bugs
- Include personal/real data
- Make false claims about features
- Use low-resolution images
- Overcrowd with text
- Show competitor apps

## üñºÔ∏è Screenshot Enhancement Options

### Option 1: Pure Screenshots
- Just the app screens, no embellishments
- Clean, honest representation
- Faster to create and update
- Good for apps with strong UI

### Option 2: Framed Screenshots
- Add device frame around screenshot
- Provides context and polish
- Tools: Frame.io, Mockuphone, Figma
- Slightly larger file size

### Option 3: Annotated Screenshots
- Add text overlays highlighting features
- Use brand colors for text boxes
- Keep text minimal (max 5-7 words)
- Good for complex features

### Option 4: Sequential Screenshots
- Show step-by-step process
- Great for demonstrating workflows
- Example: Scan ‚Üí Extract ‚Üí Save

## üìù Text Overlay Guidelines

If adding text to screenshots:

### Text Characteristics:
- **Font:** Inter, Roboto, or SF Pro (clean sans-serif)
- **Size:** Large enough to read on mobile (min 40px)
- **Weight:** Bold or Semi-bold
- **Color:** High contrast with background
- **Background:** Semi-transparent overlay for readability

### Text Content:
- **Keep it short:** 3-7 words maximum
- **Focus on benefit:** Not just feature name
- **Use active voice:** "Skenuj fakt√∫ry" not "Skenovanie fakt√∫r"
- **Localize:** Slovak for sk-SK, English for en-US

## üìä Screenshot Performance Tips

### Order Matters:
- First 2-3 screenshots are most important
- Users rarely scroll past the 4th screenshot
- Front-load your best/most unique features

### Visual Appeal:
- Use vibrant, realistic data
- Show success states, not empty states
- Include some visual variety
- Balance text-heavy and visual screens
